Accessibility



Accessibility is a critical aspect of web design, ensuring that all users, including those with disabilities, can access and interact with the content. In the Splash Screen implementation, several techniques were used to address accessibility:



1. Text Readability and Contrast

High Contrast Text: The text color (color: white;) is set against a dark background (using a gradient overlay on the background image). This ensures high contrast, making the text easily readable for users with visual impairments.

Text Shadow: The text-shadow property is applied to the main title (h1) to enhance visibility further, especially against complex background images.

2. Responsive Design

Media Queries: The use of media queries ensures that the splash screen is responsive and adapts to different screen sizes. This is particularly important for users who access the site on mobile devices or with zoomed-in views.

Example: On smaller screens (max-width: 768px), the font sizes are reduced to ensure the text remains readable without requiring horizontal scrolling.

3. Keyboard Navigation

Simple Layout: The splash screen has a simple layout with no interactive elements other than the automatic redirect. This makes it easy for keyboard-only users to navigate, as there are no complex interactions or forms that require keyboard focus management.

Automatic Redirect: The automatic redirect to the home page after 3 seconds is implemented using JavaScript (setTimeout). While this is not directly related to keyboard navigation, it ensures that users are quickly moved to the main content of the site.

4. Alternative Text for Images

Background Image: The background image is purely decorative and does not convey essential information. Therefore, it does not require alternative text. However, the text content on the splash screen is clearly visible and provides all necessary information.

Loader Animation: The loader animation is created using CSS and does not require alternative text, as it is a visual indicator of the page loading process.

5. Semantic HTML

Use of Semantic Tags: Semantic HTML tags such as h1, h2, and p are used to structure the content. This helps screen readers and other assistive technologies understand the content hierarchy and read it in a logical order.

h1: The main title of the page, which is the most important heading.

h2: A subheading for the group members section, providing a clear hierarchy.

p: Paragraphs for descriptive text, including the website's mission and group member names.



6. Focus on Simplicity

Minimalist Design: The splash screen is designed to be simple and straightforward, with minimal text and no complex interactions. This reduces the cognitive load for users with cognitive disabilities and makes the content easier to understand.

Clear Purpose: The purpose of the splash screen is clearly communicated through the text content, ensuring that users understand what the website is about before being redirected to the home page.

7. Animation and Timing

Fade-In Animations: The fade-in animations for the text content are subtle and do not flash or move rapidly, which could be problematic for users with vestibular disorders or epilepsy.

Timing: The animations are timed to ensure that the text appears gradually, giving users enough time to read the content before the automatic redirect.

8. JavaScript Considerations

Graceful Degradation: The automatic redirect is implemented using JavaScript, but the page remains functional even if JavaScript is disabled. Users will still see the splash screen content and can manually navigate to the home page if needed.

Summary

The Splash Screen implementation addresses accessibility through:

High contrast text and background for readability.

Responsive design to ensure usability on all devices.

Simple layout and semantic HTML for easy navigation and understanding.

Subtle animations and timing to avoid causing discomfort for users with certain disabilities.

Clear and concise content that communicates the purpose of the page effectively.

By incorporating these accessibility considerations, the Splash Screen ensures that it is usable and accessible to a wide range of users, including those with disabilities.

Challenges and Lessons Learned

During the implementation of the Splash Screen, Volunteer Page, and Content Page for the SDG 3 website, several challenges were encountered and resolved, leading to valuable lessons. For the splash screen, synchronizing the text animation with the loader's timing was initially problematic, resulting in abrupt transitions, but this was resolved by using setTimeout() and CSS keyframe animations, which taught the importance of managing transitions effectively for a smooth user experience. On the volunteer page, ensuring the responsive display of volunteer cards on mobile devices posed a challenge as the cards did not resize properly. This was addressed by using a flexbox layout with horizontal scrolling, along with media query adjustments, highlighting the value of testing layouts across devices. The content page faced layout issues with the navigation menu, particularly in maintaining proper alignment and avoiding overflow on larger screens. A fixed sidebar and smooth scroll behavior were implemented to resolve this, emphasizing the importance of designing intuitive user interfaces that enhance both aesthetics and usability. These experiences underscored the need for careful planning, testing, and refinement to achieve a polished, user-friendly website across different platforms.
