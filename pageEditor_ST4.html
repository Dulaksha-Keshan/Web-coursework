Profile Page


Technical Description

This HTML document represents a website dedicated to **Universal Learning Access**, specifically aligned with **SDG 3: Good Health & Well-being**. It includes a structured navigation menu that allows users to access various pages such as **Home, Volunteer, Statistics, User Profile, Feedback, Sitemap, and Team**. The page features an **interactive SVG-based sitemap**, visually displaying the website structure with clickable nodes for easy navigation. The **styling is handled via CSS**, ensuring a visually appealing interface with a background image, responsive design, and smooth scrolling behavior. Additionally, a **footer section** provides quick links, social media icons, and an "About Us" section to offer more information about the initiative. The page is designed to be user-friendly, informative, and accessible, aligning with sustainable development goals..
It also includes a progress tracker for user goals and a contact form.


Key HTML Tags:


  • <section>: Used to structure different parts of the profile, such as personal details, calendar, and goals.
  • <img>: Displays the user’s profile photo.
  • <table>: Used for the availability calendar.
  • <form>: Collects user input for contacting the profile owner.
  • <div>: Used for layout and styling purposes, such as the progress bar container.

Key CSS Properties:

  • flexbox: Ensures a responsive layout for the profile sections.
  • border-radius: Rounded corners for the profile photo and sections for a modern look.
  • box-shadow: Adds depth to the profile sections, making them visually distinct.
  • transition: Smooth hover effects on interactive elements like buttons and sections.

Purpose:

  • The <section> tags organize the content into logical parts, making it easy for users to navigate.
  • The <table> provides a clear view of the user’s availability, while the <form> allows visitors to contact the user.
  • The flexbox layout ensures the page is responsive and adapts to different screen sizes.

Accessibility

Accessibility was addressed by:

  • Adding alt text to images for screen readers.
  • Using semantic HTML (e.g., <section>, <table>) to improve screen reader compatibility.
  • Ensuring proper contrast between text and background colors for readability.
  • Making the form and calendar keyboard-navigable for users who rely on keyboard input.

Sitemap Page


Technical Description


The Sitemap Page provides an interactive overview of the website’s structure, using an SVG-based diagram to display the relationships between different pages.

Key HTML Tags:

  • <svg>: Used to create the interactive sitemap diagram.
  • <rect>: Represents the nodes (pages) in the sitemap.
  • <text>: Displays the names of the pages within the nodes.
  • <line>: Connects the nodes to show the relationships between pages.
  • <a>: Wraps nodes to make them clickable links.

Key CSS Properties:

  • fill: Sets the background color of the nodes.
  • stroke: Defines the border color of the nodes and lines.
  • text-anchor: Ensures text is centered within the nodes.
  • hover effects: Adds interactivity to the nodes, changing their appearance when hovered over.

Purpose:

  • The <svg> element allows for a scalable and interactive sitemap diagram.
  • The <rect> and <line> elements visually represent the structure of the website.
  • The <a> tags make the sitemap interactive, allowing users to navigate directly to pages.

Accessibility

Accessibility was addressed by:

  • ARIA Labels: Nodes and lines include ARIA labels for screen readers.
  • Keyboard Navigation: The sitemap is navigable using the keyboard.
  • Contrast: High-contrast colors ensure readability for users with visual impairments.

Challenges and Lessons Learned


Challenges:


  1. Responsive Design: Ensuring the profile page and sitemap worked seamlessly across devices required extensive testing and adjustments.
  2. SVG Interactivity: Making the sitemap interactive while maintaining accessibility was challenging but improved with ARIA labels and keyboard navigation.
  3. Form Validation: Ensuring the contact form on the profile page was user-friendly and accessible required careful implementation.

Lessons Learned:

  • Flexbox and Grid: These CSS tools are essential for creating responsive layouts.
  • SVG for Diagrams: SVG is a powerful tool for creating scalable and interactive diagrams.
  • Accessibility: Proper use of ARIA labels and semantic HTML greatly improves user experience.

Compliance


The pages comply with Janet regulations by:

  • Ensuring all content is accessible to users with disabilities.
  • Using semantic HTML for better search engine optimization (SEO).
  • Providing clear navigation and consistent styling across all pages.
