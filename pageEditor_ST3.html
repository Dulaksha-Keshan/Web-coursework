Feedback Page


Technical Description

The Feedback Form page is utilized to collect user feedback regarding their health and wellbeing. It is a structured HTML form with multiple input fields such as text boxes for personal details, dropdowns for selecting health ratings, checkboxes for selecting health support options, and radio buttons for user selection. Additionally, a star rating system provides an option to rate services, and a live character-counting textarea permits them to comment more. Page structure is developed with a fieldset and legend in order to group sections in a tidy fashion, thus offering a tidy and user-friendly interface.
For the sake of visual enhancement, CSS styling is used and blurred-glass effect is obtained through backdrop-filter to give a modern and visually enhancing interface. The form fields are appropriately styled with padding, borders, and hover effects to make the page visually appealing and user-friendly. Error messages appear in red color and get dynamically updated to provide users real-time feedback when fields are not filled in correctly. JavaScript is used to validate forms so that all the mandatory fields are filled in prior to submission. It also updates the character counter of the comment box in real-time and prevents submission of the form if there are any invalid mandatory fields.


Accessibility

1. Sementic HTML
The tags like header, main, footer, and fieldset enhances accessibility by providing a understandable structure for screen readers. Proper use of label elements ensures that form inputs are associated wth thair labels, making the form more accessble.
2. Keyboard Navigation
All form fields, text inputs, radio buttons and checkboxes are accessible and focusable and thus can be reached through the keyboard. This allows keyboard users to fill in the form succesfully.
3. Aria Roles and Attributes
Using aria-live="polite" on error messages will make screen readers speak out updates but not in a distracting manner to the user, thus improving the experience of visually implaired users.
The color scheme does appear to have sufficient contrast for the majority of items so that it would be simpler for visually challenged users to read the text.All input elements of the form have corresponding label elements, and error messages are displayed dynamically and hence screen reader-readable.

Team Page


Technical Description

semantic tags such as header, nav, main, and footer to provide a logical and consistent structure. The team section is structured with a.team-container and individual.team-member divs containing an image, name, and details of their role and contribution. The.details section is initially hidden and shown on hover using CSS, giving a dynamic and interactive effect on the page. The CSS styling ensures a visually appealing design. The .team-container uses display: flex with flex-wrap: wrap to create a responsive grid of team members, while the .team-member divs have a fixed width of 250px and smooth hover effects, including scaling up (transform: scale(1.1)) and reducing opacity (opacity: 0.7). The .details section becomes visible on hover, providing additional information about each team member. Typography is well-handled, with the h1 heading styled in green (rgb(14, 88, 26)) and the .name class using bold text for emphasis. Proper spacing is maintained with gap: 80px in the .team-container.
flex-wrap: wrap property makes the team members wrap to the next line on smaller screens. But more media queries would make the layout even more responsive for smaller screens.team-member and.details hover effects make the user interface interactive and engaging, thus making the team section functional and aesthetically pleasing.


Accessibility

There is an alt attribute on all the team member photos, which tells the screen reader who is in the photographs for people who are visually impaired. The navigation links and footer are also accessible via keyboard, meaning keyboard-only users are able to utilize the page. There is still room for improvement, though. The.team-member and.details hover effects are not keyboard-accessible since they are only triggered by mouse use. Including:focus styles and tabindex="0" on.team-member would also make these effects keyboard-accessible. Additionally, ARIA attributes such as aria-label or aria-labelledby can be included to give additional context to screen reader users.

Content Page


Technical Description

Employs semantic HTML elements such as header, nav, main, and footer to create a logical and easy-to-understand structure. The nav element holds anchor links that enable users to jump to the page's respective sections via # IDs, and all content is enclosed within a section element and includes a heading (h2), paragraph text (p), and an image (img). The footer has a.footer-container with "About Us," "Quick Links," and "Social Media" links, and a copyright message. The body uses flex-direction: column to vertically align content, and the nav is fixed to the left of the page with position: fixed and height: 100vh, creating a sidebar effect. The content is nudged with margin-left: 220px to accommodate the fixed navigation. Media queries adjust the layout for small screens (max-width: 768px), converting the vertical navigation to horizontal and overriding the fixed positioning. Headings (h1, h2) are accentuated with bold fonts, shadows, and borders through CSS, and paragraphs (p) include a line-height of 1.8 for legibility. The interactive items like navigation links with hover states (background-color, transform: scale(1.05)) and a "Go to Top" link (.go-to-top) with smooth scrolling (scroll-behavior: smooth) enhance the overall user experience.


Accessibility

the utilization of semantic HTML elements such as header, nav, main and footer that give a clear layout for screen readers. All images include an alt attribute, which enables screen readers to read aloud for the visually impaired. Navigation links and the "Go to Top" button are operable with the keyboard so that users of keyboard input can utilize the page.

Challenges and Lessons Learned


1. One difficulty I was encountering in creating the feedback form was showing the remaining character count dynamically for the comment field. I first attempted to accomplish this with CSS, but quickly realized that CSS was not able to monitor the user's input in real time. To achieve this, I employed JavaScript and updated the remaining character count as the user types. This lab taught me that certain things need JavaScript and not CSS and made me realize the usefulness of event handling in JavaScript more.
2. Another issue was with dropdown menus. There was always a default option selected, which can cause users to not notice that they need to make a selection. To assist in resolving this, I changed the dropdown so that no option was selected by default, encouraging users to make an active selection. This enhanced usability and user-friendliness of the form.

Compliance


Janet guidelines govern web page publishing by making websites secure, accessible, and suitable for users. My feedback form is compliant with these guidelines in a number of ways.To begin with, the page has suitable and ethical content since it is regarding acquiring user feedback on health and fitness. There is no offensive, deceptive, or injurious content, hence making it comply with content policy.
Secondly, accessibility features are met by using correct HTML structure, which comprises labeled form fields, intelligible instructions, and a coherent layout. Screen reader usability is supported by correct labels, legends, and error messages that are dynamically changed for users with disabilities. The page can also be navigated with a keyboard, hence being usable by mouse-inaccessible users.
From a data protection and privacy point of view, the form does not save or disclose user data, as required by laws prohibiting unauthorized data gathering. Although the form does not have backend storage at present, if the form were processing data, further security practices like encryption and secure data management would be justified. Above all, the page complies with some of Janet regulation's foremost demands by promoting moral content, web accessibility, users' privacy, as well as effective web composition.
